   → Muestra: Email, horario, opciones de contacto
   → Follow-ups: [Enviar mensaje privado]

📬 Ver Comunicados
   Botón: "Ver comunicados"
   → Muestra: Últimos comunicados (leído/sin leer)
   → Follow-ups: [Leer detalles]


DOCENTE (docente.html):
───────────────────────
⏰ Toma de Asistencia
   Botón: "Marcar asistencia"
   → Muestra: Lista de 20 alumnos (presente/ausente)
   → Follow-up: [✓ Confirmar]
   → Notifica a padres de ausentes

📢 Comunicados
   Botón: "Enviar comunicado"
   → Pregunta: ¿Qué quieres comunicar?
   → Genera: Borrador con tono institucional
   → Follow-up: [✓ Enviar a 25 familias]

📝 Carga de Calificaciones
   Botón: "Cargar notas"
   → Muestra: Tabla de notas editables
   → Alerta: Si alumno bajó significativamente
   → Follow-up: [✓ Guardar y notificar]

📔 Observación Pedagógica
   Botón: "Registrar observación"
   → Pregunta: ¿Qué observaste del alumno?
   → Ejemplo: "Muy participativo en clase"
   → Almacena para generar informe trimestral

🤖 Crear Actividad
   Botón: "Crear actividad"
   → Pregunta: Describe la actividad (ej: "trivia sobre fotosíntesis")
   → Genera: 8 preguntas con opciones
   → Follow-up: [✓ Publicar]

📋 Ver Informe Trimestral
   Botón: "Ver informe"
   → Muestra: Borrador del informe por alumno
   → Información: Desempeño, habilidades, áreas de mejora
   → Follow-up: [✓ Confirmar]


ADMIN (admin.html):
───────────────────
📊 Dashboard de Morosidad
   Botón: "¿Cuál es el estado de morosidad?"
   → Muestra: Total deuda, distribución por días, top familias
   → Follow-ups: [📧 Enviar recordatorios] [💳 Generar planes pago]

   → Follow-ups: [🤝 Agendar reunión] [📬 Enviar recordatorio]

   → Muestra: Pagos confirmados, proyección, comparativas

   → Muestra: % por día, comparativas con semanas anteriores
   → Follow-up: [Comparar con período anterior]

🔧 CÓMO AGREGAR NUEVOS CDUs (Para developers)

1. Abre padre.html (o docente.html, admin.html)
2. Busca "const cduResponses = {"
3. Agrega un nuevo CDU:

const cduResponses = {
  nuevo_cdu: {
    user: "Texto que ve el usuario",
    assistant: "Respuesta del asistente",
    followUp: [
      { label: "Opción 1", cdu: "otro_cdu_existente" },
      { label: "Opción 2", response: "Respuesta directa" }
    ]
  },
  ...más CDUs
}

4. Agrega un botón en el HTML:
<button class="chip chip-vertical"
        onclick="cduPadre('nuevo_cdu')">
  🎯 Nombre del CDU
</button>

5. ¡Listo! El nuevo CDU es funcional.
